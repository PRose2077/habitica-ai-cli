# Habitica-Soul 产品需求文档 (PRD)

**项目代号：** Habitica-Soul
**定位：** AI 驱动的极客级 Habitica 命令行增强工具，旨在将任务管理升华为具有叙事深度和极度可执行性的游戏化体验。

---

## 1. 核心愿景
通过 AI 介入，解决 Habitica 任务过于“空泛”且缺乏“即时正反馈”的痛点。利用**极其明确的任务拆解**保障执行力，利用**语义进化的称号系统**提供成就感，利用**黑化/心魔机制**提供负向约束。

---

## 2. 功能模块详细说明

### 2.1 智能任务拆解 (Smart Decomposition)
*   **功能描述**：使用 LLM 将模糊的任务标题转化为可立即执行的步骤。
*   **关键特性**：
    *   **动词驱动**：AI 拆解出的子任务必须以具体的“动作词”开头，并对动词进行 CLI 高亮处理。
    *   **难度映射**：AI 根据拆解后的子任务数量和复杂度，自动通过 API 设置任务的难度等级（Trivial/Easy/Medium/Hard）。
    *   **依赖链识别**：在子任务间建立逻辑先后顺序，在详情视图中以符号（如 `->`）标识。
    *   **深度控制**：提供 `--depth [1-3]` 参数，控制 AI 拆解的详细程度。
    *   **原生集成**：所有拆解步骤直接同步至 Habitica 的 **Checklist（子任务）** 系统。

### 2.2 游戏化风格引擎 (Gamification Engine)
*   **预设风格**：提供赛博朋克、修仙、硬核军事、职场讽刺等预设风格。
*   **自定义风格**：用户可通过 `custom_theme.yaml` 定义自己的世界观（含 System Prompt、动作词映射、语气风格、前缀括号样式）。
*   **全域影响**：
    *   **任务描述**：将现实任务转化为风格化的“剧情指令”。
    *   **子任务名**：Checklist 每一项都根据风格重写（保留现实目标）。
    *   **Category Key**：AI 自动为每个任务分配一个分类键（如 Coding, Health, Life），用于称号进化。

### 2.3 称号与成就系统 (Achievement Wall)
*   **语义进化（转职）**：称号不再是简单的数字等级，而是基于 AI 生成的语义晋升。
    *   *示例：[代码拾荒者] -> [二进制猎手] -> [算法架构师]*。
*   **称号记录逻辑**：
    *   **静默更新**：每晚定时脚本扫描已完成任务，计算转职进度并更新本地数据库。
    *   **历史保留（方案2）**：即使更换了风格，旧风格获得的称号依然保留在成就墙中。
*   **展示机制**：
    *   **单称号前缀**：CLI 提示符前显示当前佩戴的称号（支持手动更换）。
    *   **ASCII 艺术墙**：统一布局的成就陈列柜，展示各分类下的最高军衔、等级和获得时的风格。

### 2.4 任务调整与交互 (Interactive CRUD)
*   **方案 A（默认命令模式）**：
    *   支持 `sub-add`, `sub-done`, `sub-mv`, `task-set` 等快捷参数操作。
*   **方案 B（TUI 交互模式）**：
    *   通过 `habitica edit -G` 进入。
    *   提供类似 Vim/Tmux 的全键盘操作界面，用于批量调整子任务顺序、修改文字和配置元数据。
*   **展示逻辑**：
    *   `list` 视图：仅展示任务标题和 Checklist 进度条（如 `[====> ] 50%`）。
    *   `show` 视图：展示游戏化全文、完整子任务列表及黑化状态。

### 2.5 黑化与心魔系统 (The Corruption System)
*   **负向反馈闭环**：每日定时脚本检查过期任务。
*   **腐烂机制**：
    *   **Level 1 (过期1-2天)**：任务描述变得急躁。
    *   **Level 2 (过期3-5天)**：描述转为压抑/走火入魔状态。
    *   **Level 3 (一周以上)**：任务彻底“黑化”，描述带有挑战性或绝望感。
*   **前缀联动**：身份前缀会随之变色（如变紫/暗红），称号前自动增加“走火入魔的”、“虚弱的”等修饰词。
*   **净化收益**：完成黑化任务会获得特殊的“净化”动效及专属稀有称号。

## 3. 技术架构要点

### 3.1 数据存储
*   **本地缓存**：存储任务元数据、成就历史、当前佩戴称号。
*   **配置文件**：`config.yaml`（API Key、偏好设置）与 `theme.yaml`（风格定义）。

### 3.2 API 集成
*   **Habitica REST API v3**：深度使用 `/tasks/user` 的 PUT、POST 和 DELETE。
*   **LLM 接口**：支持 OpenAI/Anthropic/本地大模型，用于拆解任务和生成称号。

### 3.3 自动化脚本
*   **Cron Job**：每晚 0 点运行，执行以下逻辑：
    1.  扫描已完成任务 -> 触发 AI 进化称号 -> 更新本地成就墙。
    2.  扫描过期任务 -> 触发 AI 修改任务描述（黑化逻辑） -> 同步至服务端。

## 4. 关键交互示例

**场景：添加一个新任务**
```bash
$ habitica add-smart "重构数据库连接池" --style cyber --depth 2

[System] AI 正在注入灵魂...
[Style] Cyberpunk / 黑客模式已激活
[Task Created] ID: p9k2
[Difficulty] Hard (由 AI 根据 6 个子任务自动评估)
[Summary] 
- 任务：«指令» 强行渗透并重构核心数据矩阵的连接协议 (数据库连接池)
- 子任务进度：[>         ] 0%
```

**场景：查看成就墙**
```bash
$ habitica wall

/====================================================================\
|                      HALL OF ACHIEVEMENTS                          |
|====================================================================|
|  CATEGORY      |   CURRENT RANK              |  LVL  |    STYLE    |
|----------------|-----------------------------|-------|-------------|
|  Coding        |   [ 数字造物主 ]             |  15   |  Cyberpunk  |
|  Health        |   [ 筑基期大圆满 ]           |  09   |  Wuxia      |
\====================================================================/
```
